ProbablyFresh: полный запуск с нуля (PowerShell, Docker-only)

1) Открыть папку проекта
cd F:\DE_intern\probablyfresh-analytics-platform

2) Подготовить env (один раз)
copy .env.example .env
Проверь, что в `.env` задан `PII_HASH_SALT` (>=16 символов), иначе producer перейдет на fallback-соль и выдаст warning.

3) Собрать app-образ (один раз, либо после изменений requirements/Dockerfile)
docker compose --env-file .env build app

4) Полный сброс инфраструктуры и данных
docker compose --env-file .env down -v

5) Поднять сервисы
docker compose --env-file .env up -d zookeeper kafka mongodb clickhouse app grafana airflow-postgres airflow

6) Дождаться готовности ClickHouse (обязательно)
do { Start-Sleep -Seconds 2; docker compose --env-file .env exec -T clickhouse clickhouse-client --query "SELECT 1" } while ($LASTEXITCODE -ne 0)

7) Инициализировать RAW-слой ClickHouse (Kafka Engine + MV + RAW таблицы, включая purchase_items_raw)
Get-Content docker/clickhouse/init/01_init.sql -Raw | docker compose --env-file .env exec -T clickhouse clickhouse-client --multiquery

8) Инициализировать MART-слой ClickHouse (включая purchase_items_mart и mart_quality_stats)
Get-Content docker/clickhouse/init/02_mart.sql -Raw | docker compose --env-file .env exec -T clickhouse clickhouse-client --multiquery
Важно: `02_mart.sql` выполняется только после `01_init.sql`.
Если менялись `01_init.sql` или `02_mart.sql`, сначала делай `docker compose --env-file .env down -v`, затем повторяй init.

9) Сгенерировать демо JSON
docker compose --env-file .env run --rm app python src/generator/generate_data.py

10) Загрузить JSON в MongoDB
docker compose --env-file .env run --rm app python src/loader/load_to_mongo.py

11) Опубликовать Mongo -> Kafka (--once)
docker compose --env-file .env run --rm app python src/streaming/produce_from_mongo.py --once

12) Дать ClickHouse время дочитать Kafka
Start-Sleep -Seconds 5

13) Обновить snapshot качества в MART (повторный запуск 02_mart.sql)
Get-Content docker/clickhouse/init/02_mart.sql -Raw | docker compose --env-file .env exec -T clickhouse clickhouse-client --multiquery

14) Запустить ETL в S3 (Spark из app контейнера, без повторной загрузки pyspark)
docker compose --env-file .env run --rm app spark-submit --master local[*] --jars /opt/jars/clickhouse-jdbc-0.9.6-all-dependencies.jar jobs/features_etl.py
